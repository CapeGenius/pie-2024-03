OneWheel

OneWheel is a single-wheel ​electric vehicle that self-balances with a weight on top. 

Inspired by existing one-wheel devices like hoverboards,
our OneWheel Project is a hands-on learning experience where
we set out to develop a self-balancing, single-wheel electric vehicle.

We applied engineering concepts and tackled real-world challenges while deepening our understanding of control systems, mechanical design, and teamwork through the process of building a functional device that stands out for its stability and versatility.

Key Features
Precision Control: ​A PID controller ensures the robot maintains balance and adjusts to disturbances.
Smart Adaptation: Sensors monitor rider input, enabling smooth and responsive control.

System Diagram

Schematics

  • The Arduino Uno serves as the brain of the system, reading data from the IMU and sending control signals (PWM) to the motor driver.
  • The motor driver controls motor speed, direction, and braking using the input signals from the Arduino.
  • The Hall sensor feedback provides critical motor positional data for smoother control.

System Diagram - Components and Flow

This system diagram provides an overview of the flow of components in the OneWheel project.

01

Power Sources

  • 9 V Battery: Supplies power to the Arduino.
  • Lithium-Ion Battery (39V): Powers the VESC/Motor Controller and the hub motors.
02

Arduino Uno R3

  • Acts as the central controller of the system.
  • Receives commands from an external program to control the system.
  • Receives angle values from the IMU sensor, helping determine the tilt angle of the system.
03

IMU

  • (Inertial Measurement Unit)
  • A sensor providing angle values to the Arduino.
  • Helps the system maintain balance by determining the tilt of the OneWheel.
04

VESC/Motor Controller

  • Receives speed values from the IMU and Arduino to control the motor’s speed.
  • Receives PWM signal generated to manage motor power output.
  • Drives Hub Motors.
05

Hub Motors

  • The Hub Motors are responsible for driving the OneWheel.
  • They interact with the Metal Frame.
  • Hall Sensor Information: The motors send back speed and positional data to the VESC/Motor Controller, enabling feedback-based control.
06

User Input

  • A "user" interacts with the system by affecting the balance. The PWM signal and control system adjust accordingly to maintain stability.

Schematics - Components and ​Connections

The detailed hardware wiring schematic shows connections between components.

01

Motor Driver

  • (VESC/Motor Controller)
  • The motor driver regulates power to the BLDC Motor using:
  • Motor Phase Wires (MA, MB, MC): Connected to the motor's coils for control.
  • Hall Sensor Wires (Hc, Hb, Ha): Provide motor speed and position feedback.
02

Arduino Uno R3

  • Receives 5V power from the battery.
  • Speed (Pin 12): Reads or sends speed-related signals to monitor or adjust motor performance.
  • PWM (Pin 9) : Sends PWM signals to the motor driver to control motor speed.
  • DIR (Pin 2): Controls motor rotation direction.
  • Brake (Pin 7): Activates braking. A switch is connected.
03

IMU

  • The IMU sensor is wired to the Arduino for balance control:
  • VCC/GND: Power supply for the IMU.
  • SCL/SDA: I²C communication lines for data exchange.
04

Motor (BLDC)

  • Receives power from the motor driver through motor phase cables.
  • Sends Hall sensor feedback for accurate control.
